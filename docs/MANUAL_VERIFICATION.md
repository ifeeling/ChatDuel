# ChatDuel v1 端到端验收清单

> 跑通下面所有项即认为 v1 达成。
> 涉及真实 ChatGPT / Gemini 的项目需要用户登录 + 在浏览器里手动验证。

## 准备
- [ ] `npm install` 完成
- [ ] `npm run build` 成功生成 `dist/`
- [ ] `npm test` 通过（40 个单元测试）
- [ ] `npm run test:e2e` 通过（mock PoC）

## 加载扩展
- [ ] 打开 `chrome://extensions`
- [ ] 开启"开发者模式"
- [ ] "加载已解压的扩展" → 选择 `dist/` 目录
- [ ] 扩展出现在列表里，名称"ChatDuel"

## 真实页面验证
- [ ] 打开 `https://chatgpt.com`，登录
- [ ] 打开 `https://gemini.google.com`，登录
- [ ] 点击扩展图标，弹出 popup
- [ ] popup 顶部状态条显示"ChatGPT: ✓"和"Gemini: ✓"
- [ ] 在输入框输入"你好"，点击"发送"
- [ ] 等待两边都出现"Mock AI response..."字样（如使用 mock）或真实回答
- [ ] 两边回答下方出现 stats 行（字数、耗时、首次 Token 时间）

## @ 选择性发送
- [ ] 在输入框输入 `@chatgpt 你好`
- [ ] 点击发送
- [ ] 只有 ChatGPT 那侧出现用户气泡，Gemini 完全静默

## 双方答完视觉提示
- [ ] 当两边都完成时，两侧面板都高亮闪烁一次

## 搬运
- [ ] 等两边都完成后，点击 "C→G" 按钮
- [ ] Gemini 侧出现"下面是另一个 AI 的回答..."开头的用户气泡
- [ ] Gemini 给出对 ChatGPT 回答的审查

## 引用
- [ ] 等两边都完成后，点击"引用"按钮
- [ ] 输入框被插入"[引用 ... 的上一条回答]："开头的文本

## 对比总结
- [ ] 点击"对比总结"
- [ ] ChatGPT 侧收到包含双方回答的总结请求

## 字符串 diff
- [ ] 当两边都完成时，回答面板里的不同部分有绿/蓝色高亮
- [ ] 双方都提到的部分没有背景色

## 快捷键
- [ ] 在输入框按 `Ctrl/Cmd+Enter` → 发送
- [ ] 在输入框按 `Ctrl/Cmd+Shift+1` → 触发 C→G
- [ ] 在输入框按 `Ctrl/Cmd+Shift+2` → 触发 G→C

## 图片输入
- [ ] 复制一张图片到剪贴板
- [ ] 在输入框按 `Ctrl+V`
- [ ] 两边都收到图片（成功路径）
- [ ] 或失败时显示"下载图片"按钮（兜底路径）

## 本地历史
- [ ] 多次发送问题后点击"历史"
- [ ] 看到刚才的会话列表
- [ ] 输入序号查看详情
- [ ] 选择删除一条

## 设置页
- [ ] 右键扩展图标 → 选项
- [ ] 看到"启用的 AI 平台"开关
- [ ] 看到两个 adapter 的版本号和最后验证时间
- [ ] 看到"已保存 N 条会话"统计
- [ ] 取消勾选 Gemini，popup 中 Gemini 面板禁用

## 错误处理
- [ ] 等 ChatGPT 出现"继续生成"按钮时，主页面应提示"⏸ ChatGPT 暂停"
- [ ] 在 ChatGPT 快速发 5+ 条消息触发限流时，主页面应显示"⚠️ 限流"

## 稳定性
- [ ] 让扩展在后台闲置 5+ 分钟，再打开 popup，状态应正确恢复
- [ ] 关闭 ChatGPT 标签页，popup 应提示"标签页已关闭"
- [ ] 真实场景连续使用 30 分钟无崩溃
